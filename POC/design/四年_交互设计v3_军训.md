# 军训场景 · 交互设计 v3（节奏游戏 + 场景探索）

> **设计哲学**：不是"呈现选项让你选"，而是"你探索场景，选择自然浮现"。
> 
> **核心原则**：无论用户做了什么操作，画面都要有对应变化。没有"无效点击"。
> 
> **v3新增**：选择3（阅兵领队）引入节奏小游戏——自荐后不是叙事交代结果，而是玩家真的"走两步"。

### ⭐ 节奏小游戏：走正步

**触发条件**：玩家选择自荐后进入

**视觉设计**：
```
   ┌──────────────────────────────┐
   │                              │
   │   🚶 (你的简笔画，侧面)      │
   │   ↙  ↘  ↙  ↘              │
   │                              │
   │  ─ ─ ─ ─ ─ ─ ─ ─ ─ ─      │  ← 节拍线，从右向左滚动
   │          ↓                   │
   │      【击！】                │  ← 打击区
   │                              │
   │   得分: ●●●○○○○  连击: 3    │
   │   进度: ████░░░░░░  8/16    │
   └──────────────────────────────┘
```

**玩法**：
1. 画面变成侧面视角，你的角色在"走正步"的简笔画动画
2. 节拍线从右向左匀速滚动，经过"击"标记时需要点击屏幕
3. 共16个节拍，模拟正步走的一个完整方队通过
4. 每次点击判定：
   - 完美（Perfect）：角色步伐有力，画面闪一下
   - 良好（Good）：还行但不够整齐
   - 错过（Miss）：角色踉跄一下，其他排有人偷笑
5. 底部实时显示得分和连击数

**判定标准**：
- 完美/良好 ≥ 12次 → 你被选上了 ✅
- 完美/良好 < 12次 → 你落选了 ❌（教官选了另一个人）
- 全Perfect → 彩蛋：教官说"不错，有底子"

**视觉反馈**：
- Perfect：角色步伐整齐有力 + 屏幕微闪 + 节拍命中粒子效果
- Good：还行，稍微歪了一点
- Miss：角色踉跄/顺拐 + 旁边同学偷笑表情 + 轻微画面抖动
- 连续Perfect：角色越走越自信，画面越来越亮
- 连续Miss：角色越来越慌，旁边的同学开始偷看

**声音暗示**（如果有音频能力）：
- 节拍有轻微的"哒、哒、哒"声
- Perfect：清脆的"咔"
- Miss：沉闷的"咚"

## 节奏游戏的设计意图

1. **玩家不只是"选"，而是"证明"**：自荐不是点一个按钮，你真的要走两步
2. **努力有意义**：即使是C路径的遗憾，至少你尝试了。"你敢站出来，这比结果重要"
3. **失败也是故事**：落选不是"Game Over"，是另一条故事线的开始
4. **搞笑元素延续**：Miss时角色踉跄、同学偷笑——即使在紧张的选拔里也有笑点
5. **重玩意愿**：节奏游戏天然有"再来一次"的冲动
6. **节奏感和军训的契合**：军训最核心的体验就是"节奏"——口令、正步、口号，全是节奏

## 技术实现要点

### 节奏游戏
- 使用 requestAnimationFrame 实现平滑滚动
- 节拍间隔约800-1000ms（模拟正步速度，实际正步约116步/分钟）
- 打击窗口：Perfect ±80ms，Good ±160ms，超出=Miss
- 触摸事件监听（touchstart），支持全屏点击
- 简笔画角色用CSS动画实现走步（腿部摆动）

### 场景切换
- 每个场景用独立的div，通过display切换
- 场景间用淡入淡出过渡（500ms）
- 场景数据从JSON读取

### 通用交互
- 每个可交互元素是div，有点击计数器
- 气泡用绝对定位+CSS动画
- 选项按钮和普通气泡视觉区分（选项有更明显的边框/颜色）

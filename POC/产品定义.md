# 那个谁 - 产品定义

## 一句话定位
一款面向准大学生和大一新生的互动叙事产品。玩家通过情境选择题匹配到一个真实角色（基于真实人物原型），"穿越"成那个人，以他的身份过大学生活，体验他的选择和后果。

## 产品名
**那个谁**

## 核心理念
- **不是"你"的故事，是"那个人"的故事**：玩家不是玩"自己的大学生活"，而是"穿越"成某个特定的人，体验他的大学生活
- **真实角色原型**：每个角色都基于真实人物（包括设计者本人、他的同学、朋友），有真实的背景和经历
- **情境匹配**：玩家不是"选择"角色，而是通过情境选择题"匹配"到角色——你的选择暴露了你的性格，性格决定了你是谁
- **悬念与共鸣**：玩家会好奇"我会匹配到谁？"，玩完后会想"原来他经历了这些"

## 目标用户
- 核心：高考结束后的准大学生（6-9月）、大一新生（9-1月）
- 次要：大二大三学生（选择与自己阶段相关的场景）

## 核心玩法

### 游戏流程
1. **角色匹配**：玩家进入游戏，先完成一组情境选择题（5-8题），根据选择模式匹配到一个角色
2. **角色档案**：匹配后，玩家看到"角色档案"——这个人是谁、什么背景、什么性格
3. **穿越体验**：玩家以这个角色的身份过大学生活，场景和选择是角色特定的
4. **结果呈现**：每个场景结束后生成结果卡，包含路径、一句话总结、遗憾目录
5. **角色揭秘**：游戏结束后，揭示"这个角色是基于真实人物原型"，增加真实感

### 角色匹配系统（情境选择题）

**设计原则**：
- 不是"性格测试"，是"情境选择"——玩家在做选择，不是在回答问题
- 每道题都是一个具体的大学情境，玩家选择自己会怎么做
- 根据选择模式匹配到角色（不是数值累积，是行为模式一致性）

**题目示例**（MVP阶段5题）：

1. **室友让你帮带饭，你会？**
   - A. 行吧，顺便 → 倾向"好说话/回避冲突"
   - B. 你自己去吧 → 倾向"直接/有边界感"
   - C. 看情况，上次他也没帮我 → 倾向"计较/公平导向"

2. **这节课不点名，你去不去？**
   - A. 去，万一有重点呢 → 倾向"谨慎/积极"
   - B. 不去，反正不点名 → 倾向"放松/实用主义"
   - C. 问室友去不去，他去哪我去哪 → 倾向"从众/社交导向"

3. **小组作业有人划水，你怎么办？**
   - A. 私聊他，问问情况 → 倾向"温和/沟通导向"
   - B. 直接找老师说 → 倾向"直接/规则导向"
   - C. 算了，我帮他做了 → 倾向"妥协/回避冲突"

4. **朋友圈想吐槽，但辅导员加了你微信，你？**
   - A. 发，不管了 → 倾向"直接/不在乎评价"
   - B. 设置分组，屏蔽辅导员 → 倾向"谨慎/有边界感"
   - C. 算了不发了 → 倾向"回避/压抑情绪"

5. **喜欢的人给你发了条微信，你怎么回？**
   - A. 秒回，但装作不在意 → 倾向"积极/但害羞"
   - B. 等一会儿再回，不能显得太急 → 倾向"策略/在乎形象"
   - C. 不知道怎么回，问室友 → 倾向"依赖/社交导向"

**匹配逻辑**：
- 每个选项对应一个"行为标签"（回避/直接/温和/从众等）
- 5题结束后，统计玩家的主要行为模式
- 根据行为模式匹配到角色（比如"回避型"匹配到"那个5年半的人"）

**角色池设计**（MVP阶段4个角色）：
1. **那个5年半的人**（设计者本人）：回避型，连续妥协，最后延期毕业
2. **那个学霸**：积极型，事事争先，但压力大到崩溃
3. **那个社交达人**：从众型，人缘好但没主见，最后迷失自我
4. **那个创业者**：直接型，有主见但冲动，最后退学创业

### 场景设计
- 每周两更（周一、周四），每次一个新场景
- 每个场景是角色特定的故事线，包含多个选择节点
- 玩家做出选择 → 触发不同分支 → 看到真实的结果和后果
- 结果附带「遗憾目录」，翻开你没走的那条路上错过的东西——不说教，只呈现你可能错失的瞬间
- 某些场景可以跨角色复用（比如"百团大战"），但前置背景和选择后果是角色特定的

## 内容要求
- **独特**：每个场景必须是大学独有的体验，换任何人生阶段都不会发生
- **真实**：情节基于真实大学经历，不是编造的鸡汤
- **细碎**：不要宏大事件，要日常中那些破事——越小越真实
- **纠结**：没有"正确答案"，每个选择都有微妙的代价
- **共鸣**：让玩家觉得"靠，这不就是我昨天遇到的吗"
- **深入**：即使场景小，支线至少 3 层，选择后果要连锁

### 场景设计结构

每个场景必须包含以下结构，不能直接给选择：

1. **前置背景（情境构建）**：3-5条要点，构建"为什么会有这个选择"的前因后果。包括：
   - 人物关系（你和谁？什么关系？）
   - 时间线（之前发生了什么？）
   - 心理状态（你现在在想什么？纠结什么？）
   - 外部压力（别人怎么说的？环境怎么影响的？）

2. **情境**：1-2句话，把前置背景浓缩成"现在这一刻"的场景。

3. **核心选择节点**：基于情境的选择，每个选择都有明确的后果和连锁反应。

4. **结果卡**：路径 + 一句话 + 遗憾目录。

**关键原则**：玩家不是在真空里做选择，是在一连串前因后果下做出的决定。选择的意义来自情境，不是来自选项本身。

## 传播设计
- **角色匹配结果**：玩家匹配到角色后，可以分享"我匹配到了那个谁"，增加悬念和讨论
- **场景结果卡**：每个场景结束生成一张「结果卡」，可分享到朋友圈/群
- **结果卡内容**：包含玩家的选择路径、一句话总结、遗憾目录，有社交货币属性
- **角色揭秘**：游戏结束后揭示"这个角色是基于真实人物原型"，增加真实感和讨论度
- 微信内可直接打开（H5/小程序）

## 技术形态
- 初期：H5 页面，微信内传播
- 中期：微信小程序
- 后期：App（如果需要）

## 变现模式
- **周一更新**：全量免费，所有用户可体验
- **周四加更**：付费订阅用户专属，**9.9 元/周**
- 付费解锁后可以回看所有历史付费内容
- （远期）品牌合作：大学官方定制场景
- （远期）周边衍生：大学攻略手册、实体卡牌

### 场景3（免费·周一）：小组作业有人划水

**前置背景（情境构建）**：
- 这门课占学分很重，大作业占期末成绩的40%
- 你们是自由组队，你和另外两个人比较熟，那个划水的人是老师随机分配的
- 第一次组会时他说了句"我可以做PPT"，之后就再没在群里说话
- 你私聊过他一次，他说"在做了在做了"，但什么都没交
- 另外两个组员已经开始急了，有人在群里说"要不要找老师说"
- 你心里在想："如果我帮他做了，他反而轻松了；如果不帮，我们组可能拿不到高分"

**核心选择节点**：

1. 你怎么做？
   - 群里直接 @ 他问进度 → 他说"在做了在做了"，然后继续消失
   - 私聊他 → 他说最近家里有事，但不愿意说具体
   - 不管了，你帮他做 → 你熬夜到凌晨 3 点
   - 和另外两个人商量怎么办 → 有人说"算了我去找老师说"

2. 如果找老师说：
   - 老师说不想管你们的事 → 你们自己扛
   - 老师说可以给那个同学 D → 他来找你们求情
   - 老师把那个人调走了，但没有新人加入 → 你们 3 个人分 4 个人的活

3. 如果你们帮他做了：
   - 他来说谢谢 → 你觉得很虚伪
   - 提交时你把他名字从作者栏删了 → 他发现后在群里吵
   - 你保留了他名字，但心里不平衡 → 下次选组你再也不要和他一组
   - 你给他分配了"做PPT美化"这种边缘任务 → 至少他参与了

4. 答辩时：
   - 老师问"你们每个人负责了什么" → 那个人说了和你完全不同的版本
   - 老师直接问那个人一个细节问题 → 他答不上来，全场尴尬
   - 你们组拿了最高分 → 你觉得不公平，因为分都是你们挣的

**结果卡**：
- 路径：「私聊 → 他说在做 → 你帮他做了 → 答辩时他抢功 → 你憋了一肚子气」
- 一句话：「大学 4 年你会经历至少 6 次小组作业，每次都有人划水」

**遗憾目录**（你可能错过的）：
- 答辩时你据理力争的那条线上，老师课后私下问你"要不要加入我的课题组"——因为你有主见。你没走那条路
- 那个划水的同学后来保研了，因为这门课你帮他拿了同样的分
- 你熬夜到凌晨3点的那晚，错过了室友生日——他在群里说"算了，他忙"。你没看到
- 如果你在第一次组会就说"每个人负责的部分自己讲"，后来答辩时他抢不了功。你没选这条路

### 场景4（付费·周四）：朋友圈该不该屏蔽辅导员

**前置背景（情境构建）**：
- 大一开学时辅导员加了你的微信，说"有事随时联系"
- 你一直没设置分组，因为觉得"都是朋友"
- 这周过得很糟：思修差点挂、和室友因为带饭的事闹了别扭、小组作业还被人坑了
- 你喝了点酒（其实就两瓶啤酒），想发条朋友圈吐槽
- 写到一半你突然想起来——辅导员能看到你的朋友圈
- 你开始纠结："如果他不高兴怎么办？""如果他在年级会上点名怎么办？""但如果我屏蔽他，他会不会觉得我有心事？"

**情境**：周五晚上 11 点，你看着手机屏幕，朋友圈编辑界面里写了一段话。你的手指悬在"发送"按钮上，但你没点下去。

1. 发不发？

## 隐藏结局：五年半

> **设计理念**：隐藏结局不是"彩蛋"或"收集品"，而是对游戏主题的补充——大学不只有"精彩四年"，还有"有人走了5年半"。触发逻辑基于玩家的行为模式一致性，不是数值累积。玩家感受到的是"我的选择暴露了我的性格"，而不是"我攒够了分数"。

### 触发机制：行为模式检测

- 每个场景的选择背后有**态度标签**：回避/妥协/积极/争取
- "算了不去了""算了忍了""算了帮他做""算了不发了" → 回避/妥协
- "起来吧""直接拒绝""据理力争""发真实的" → 积极/争取
- 如果玩家在 4 个场景中有 **3 次以上** 选择了回避/妥协类选项 → 触发隐藏结局
- **关键**：玩家不知道触发条件。不是因为"数值满了"，而是因为"你的选择模式暴露了你的性格"——和真实世界的逻辑一样

### 呈现方式：不刻意提示

- 第 4 个场景结束后，结果卡正常出现
- 如果触发了隐藏结局，结果卡底部自然多出一行小字："你想知道另一个人的故事吗？"
- 玩家点击 → 进入隐藏结局
- **没有"恭喜解锁"**，没有弹窗，没有特效。叙事不能被打断

### 玩法：镜像叙事

- 隐藏结局的玩法和前面**完全一样**：选择 → 分支 → 结果
- 但内容不是"你"的故事，是"有个人"的故事
- 玩家扮演另一个角色（读了 5 年半的那个人），做出选择，看到后果
- 每个选择节点旁边有一行小字："如果是你，你会怎么选？"
- **设计意图**：
  - 保持互动性（玩家还在选）
  - 产生镜像对比（"他的选择" vs "我的选择"）
  - 情感冲击来自对比："原来他的每一步和我这么像，但结果不同"
